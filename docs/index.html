gpufab is a deterministic discrete-event simulator for GPU training fabrics. It models congestion where it actually forms — queues, drops, ECN, retransmission — and measures routing strategies against each other with paired seeds and confidence intervals. Platform

Congestion modeled where it forms, routing swapped where it counts

A single-threaded deterministic core advances packet-train events on an integer-picosecond virtual clock. Telemetry flows from the data plane to a pluggable routing interface; the closed feedback loop from queue depth to path decision is the object of study.

Deterministic core

Virtual clock in integer picoseconds keeps serialization delays exact and event order free of floating-point drift. No wall-clock, no sockets, no threads in the model path. One master seed; every stream named and logged.

Real data plane

Per-link FIFO queues with buffer limits, tail-drop, and ECN marking; a TCP-shaped transport with SACK-style fast retransmit and RTO backoff; incast, ring all-reduce, and permutation workloads. Congestion emerges, it is not scripted.

Pluggable routing

One IRouter interface: telemetry in, path out, never the wall clock. ECMP (control), a CONGA-style flowlet balancer, and a reinforcement-learning policy swap behind it with zero engine changes.

Measured results

One control, one heuristic, one experiment — reported with equal weight

64-host two-tier Clos at 4:1 oversubscription, 2 MiB uniform-permutation flows, 20 seeds. The same 20 traffic patterns are replayed for every router, so comparisons are paired; percentiles carry bootstrap confidence intervals.

Reducing exploration makes its tail worse: concurrent senders greedily sharing one learned policy herd onto the same spine. Verification & reproducibility

Engineering assurance, in place of marketing claims

Every claim on this page traces to a seeded run directory stamped with its config hash and git SHA. These are the standing guarantees.

Validated before compared

The simulator reproduced the SIGCOMM 2009 incast throughput-collapse curve before any router comparison ran. The gate is enforced by CI with a nonzero exit if the collapse disappears.

Deterministic by construction

Integer-picosecond clock, seeded named RNG streams, insertion-order event tie-breaking. Two runs with the same inputs produce byte-identical output; a test asserts it.

Sanitized on every push

AddressSanitizer and UndefinedBehaviorSanitizer builds with warnings-as-errors run the full test suite in CI, alongside an end-to-end artifact check and the validation gate.

Anchored to closed forms

A single uncontended flow’s completion time must equal the analytic store-and-forward value exactly, to the picosecond. The anchor keeps the model honest as features accrete.

Statistics, not anecdotes

Twenty paired seeds per comparison, pooled percentiles with bootstrap 95% confidence intervals, identical traffic replayed per router. Medians and tails reported side by side.

Limitations stated up front

Packet-train granularity, no PFC, single failure domain, static topology; p99.9 can regress on a minority of seeds under any rerouting scheme. Scale claims stop at 1,024 simulated hosts, because that is what was measured.

Reproduce

Every figure, one command each

Runs land in directories stamped with config hash and git SHA. A dirty working tree is recorded in the manifest and disqualifies the artifact.

# build the core (C++20, no third-party dependencies)
cmake -B build -DCMAKE_BUILD_TYPE=Release && cmake --build build -j
python3 -m venv .venv && .venv/bin/pip install -r requirements.txt

# validation gate — exits nonzero unless the collapse reproduces
.venv/bin/python python/validate_incast.py
peak=0.981 tail=0.082 collapse=YES

# three-way comparison, 20 paired seeds, bootstrap CIs
.venv/bin/python python/harness.py --routers ecmp,flowlet,rl --seeds 20

# link-utilization heatmap from any run directory
.venv/bin/python python/plot_heatmap.py results/sweep/runs/<run>
